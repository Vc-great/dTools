# 前端部署系统使用指南

## 功能概述

这是一个基于Tauri2开发的桌面端前端部署系统，支持：

1. **配置化部署**: 通过TOML配置文件管理部署参数
2. **实时日志**: 部署过程中的日志实时传递到前端显示
3. **进度跟踪**: 可视化部署进度条
4. **跨平台支持**: 兼容Windows和macOS
5. **Git集成**: 支持Git仓库克隆和分支切换
6. **SSH部署**: 通过SSH连接到远程服务器
7. **Docker容器化**: 自动构建和部署Docker容器

## 系统架构

```
前端 (React + TypeScript)
    ↓ (Tauri事件系统)
Rust后端服务
    ↓
部署流程: Git克隆 → SSH连接 → 文件上传 → Docker构建 → 容器部署
```

## 快速开始

### 1. 配置部署参数

在前端界面的TOML编辑器中配置部署参数：

```toml
[git]
repo_url = "http://your-git-repo.git"
branch = "main"
username = "your_username"  # 可选，私有仓库需要
password = "your_password"  # 可选，私有仓库需要

[vm]
host = "192.168.1.100"
user = "root"
password = "your_ssh_password"
deploy_path = "/opt/your-app"

[docker]
image = "your-app:latest"
container = "your-app-container"
port_mapping = "8080:80"
```

### 2. 开始部署

1. 点击"保存并部署"按钮
2. 系统会实时显示部署进度和日志
3. 部署完成后会显示成功或失败状态

## 部署流程详解

### 步骤1: 克隆代码 (15%)
- 从Git仓库克隆指定分支的代码
- 支持公开和私有仓库
- 自动切换到指定分支

### 步骤2: 连接虚拟机 (30%)
- 通过SSH连接到目标服务器
- 验证用户凭据

### 步骤3: 上传文件 (50%)
- 使用tar压缩传输文件（提高效率）
- 自动排除.git等不需要的文件
- 在macOS上处理扩展属性

### 步骤4: 构建镜像 (70%)
- 在远程服务器上构建Docker镜像
- 实时显示构建输出

### 步骤5: 部署容器 (90%)
- 停止并删除旧容器
- 启动新容器
- 验证容器状态

### 步骤6: 清理 (100%)
- 清理临时文件
- 完成部署

## 开发和测试

### 运行测试

```bash
# 进入Rust项目目录
cd src-tauri

# 运行所有测试
cargo test

# 运行包含网络的集成测试（需要网络连接）
cargo test -- --ignored

# 运行特定测试
cargo test test_config_validation
```

### 测试覆盖

1. **单元测试**: 配置序列化、日志消息创建等
2. **集成测试**: Git克隆功能（需要网络）
3. **性能测试**: 大量日志消息处理
4. **配置验证**: 必填字段检查

### 开发调试

```bash
# 启动开发服务器
pnpm start

# 在另一个终端启动Tauri开发模式
cd src-tauri
cargo tauri dev
```

## 跨平台兼容性

### Windows
- 支持Windows 10/11
- 使用Windows子系统处理SSH连接
- 自动处理路径分隔符

### macOS
- 支持macOS 10.15+
- 处理macOS扩展属性（xattr）
- 优化tar命令参数

### 依赖要求
- Git客户端
- SSH客户端
- Docker（在目标服务器上）
- tar命令（文件打包）

## 安全注意事项

1. **密码安全**: 建议使用SSH密钥而非密码
2. **网络安全**: 确保SSH连接使用安全端口
3. **权限控制**: 使用最小权限原则
4. **日志安全**: 避免在日志中显示敏感信息

## 故障排除

### 常见问题

1. **Git克隆失败**
   - 检查仓库URL是否正确
   - 验证网络连接
   - 确认认证信息

2. **SSH连接失败**
   - 检查服务器IP和端口
   - 验证用户名和密码
   - 确认防火墙设置

3. **Docker构建失败**
   - 检查Dockerfile是否存在
   - 验证Docker服务状态
   - 查看构建日志

4. **容器启动失败**
   - 检查端口占用
   - 验证镜像是否构建成功
   - 查看容器日志

### 日志分析

- 命令执行输出
- 错误信息
- 时间戳
